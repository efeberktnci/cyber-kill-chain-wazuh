# Contributing

This project prioritizes reproducibility, defensive value, and evidence quality.

## Contribution Requirements

- Keep changes focused and explain their operational value.
- Use professional English in published documentation.
- Follow the scenario and evidence templates.
- Test detection content against the documented lab baseline.
- Cite authoritative sources for technique mappings and product behavior.
- Never submit secrets, credentials, malware, or unsanitized sensitive evidence.

See [`docs/git-workflow.md`](docs/git-workflow.md) for branch, commit, and review conventions.
